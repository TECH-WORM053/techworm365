# Mirror Boundary Lab — Codex Prototype v0.2

> Codex 전용 작업 폴더입니다. Claude Code가 작업 중인 `prototype/ai-mirror/`와 코드·포트·의존성을 공유하지 않습니다.

## 한 문장 정의

AI가 얼굴을 더 정확히 판정하는 거울이 아니라, **사용자가 AI 해석의 범위와 낯섦을 직접 조절하는 거울**입니다.

핵심 가설:

> 사용자가 무엇을 읽힐지와 얼마나 낯설게 변환될지를 통제하면, 판단받는 불편함 없이 개인화된 경험을 만들 수 있는가?

## Claude 프로토타입과 다른 점

| 구분 | Codex — Mirror Boundary Lab | Claude 프로토타입 |
|---|---|---|
| 중심 질문 | 해석의 통제권과 낯섦의 경계 | Vision AI가 착용 사진을 어떻게 해석하는가 |
| 해석 방식 | 시각 신호 측정 + 사용자의 문장 선택 | Claude Vision의 시적 해석 |
| 얼굴 판단 | 얼굴형·감정·성별·나이 추측 안 함 | 비전 모델 해석 실험 |
| 데이터 | 브라우저 메모리에서만 처리 | 별도 Claude 폴더의 구현에 따름 |
| 외부 API | 없음 | Anthropic SDK 사용 |
| 실행 포트 | `5174` | 별도 포트 |

- 개인화 체감·불편함 5점 평가

## 실행

이 폴더에서:

```powershell
node server.mjs
```

브라우저에서 다음 주소를 엽니다.

```text
http://127.0.0.1:5174
```

또는 PowerShell 실행 정책의 영향을 피하려면 저장소 루트에서:

```powershell
cmd /c npm --prefix codex-ai-mirror-prototype run start
```

외부 패키지가 없으므로 `npm install`은 필요하지 않습니다.

휴대폰에서는 [MOBILE-DEPLOYMENT.md](MOBILE-DEPLOYMENT.md)의 같은 와이파이 촬영 방법 또는 GitHub Pages HTTPS 배포 방법을 사용합니다.

코드 문법과 서버·개인정보 조건을 검사하려면:

```powershell
node smoke-test.mjs
```

이 검사는 별도 포트 `5175`를 잠깐 열었다 닫으며, 6개 화면·보안 헤더·정적 자산·외부 전송 코드 부재를 확인합니다.

## 개인정보 처리 원칙

- 카메라 프레임을 서버로 보내지 않습니다.
- 브라우저 저장소(localStorage, IndexedDB, 쿠키)를 사용하지 않습니다.
- 결과 JSON에는 이미지 픽셀이나 얼굴 프레임을 넣지 않고, 명도·대비·색온도 같은 파생 신호와 선택값·시간만 넣습니다.
- `추상 실루엣`과 `선택한 무드`는 결과 장면에 원본 프레임 픽셀을 사용하지 않습니다.
- `인물 + 공간`은 로컬 결과 장면에 원본 픽셀을 사용하며, 사용자가 PNG 저장을 누른 경우 그 결과가 현재 기기에 포함됩니다.
- 처음으로 돌아갈 때 Canvas와 세션 값을 지웁니다.
- 저장 버튼은 사용자가 직접 누른 경우에만 현재 기기로 파일을 내려받습니다.

## 정직한 한계

이 버전은 실시간 생성 AI 결과물이 아닙니다. 실험 01에서 관찰한 `거울·금속·유리·빛` 모티프를 Canvas 규칙으로 구성한 **인터랙션 가설 프로토타입**입니다. 목적은 다음을 빠르게 검증하는 것입니다.

1. 사용자가 해석 문장을 직접 고르는 흐름을 이해하는가?
2. 낯섦 슬라이더가 통제감과 개인화 체감에 영향을 주는가?
3. 동의·비저장 안내가 얼굴 사용의 불편함을 줄이는가?

실시간 Vision/Image API는 이 가설이 사용자 테스트에서 의미가 확인된 뒤 붙입니다.
