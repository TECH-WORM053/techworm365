# exp02 — 제품 추천 프롬프트 (복사용)

가정: **일반 고객이 선글라스를 끼고 거울 앞에 섰을 때, AI 미러가 "다른 제품은 어떠신가요"라고 추천한다.**
추천 근거를 다르게 한 **2가지 버전(A: 무드 기반 / B: 하이브리드)**을 비교한다.

## 사용법

각 모델(GPT·Gemini·Claude 등)에서 **새 대화**를 열고:
1. 사진을 첨부한다 (얼굴과 착용한 선글라스가 잘 보이는 새 사진)
2. 아래 **[제품 카탈로그]** 블록을 붙여넣는다
3. 이어서 **프롬프트 A** 또는 **프롬프트 B** 를 붙여넣는다
4. 나온 답을 `outputs/`에 저장 (예: `gpt_A_01.txt`, `gemini_B_01.txt`)

> A와 B를 같은 사진·같은 모델로 각각 돌려서, "추천 근거가 결과를 어떻게 바꾸는지" 비교한다.
> 조건은 exp01과 동일하게: 새 대화, reasoning effort 최대, 각 2회.

---

## [제품 카탈로그] — A·B 공통, 먼저 붙여넣기

```
[제품 카탈로그] (이 목록 안에서만 추천할 것. 없는 제품은 지어내지 말 것)
1. Papas 01 · 가는 아세테이트 D-프레임+메탈 · 실험적·개성·얇고 도회적
2. Her 01 · 각진 스퀘어, 블랙 · 클래식·데일리·정제됨
3. Lang 01 · 라운드 플랫바, 세로폭 짧은 곡선, 스터드 · 부드럽고 미니멀·유니크
4. Liliit 01 · 오버사이즈 스퀘어, 라운딩 프론트, 사이드 메탈 · 와이드·존재감·시크
5. Pino 01 · 구조적 스퀘어, 사이드 메탈 아이콘 · 트렌디·균형
6. Jade 01 · 렌즈 작은 라운드 · 담백·절제·조용함
7. Malta 01 · 볼륨 라운드 · 럭셔리·무게감·유니섹스
```

## 프롬프트 A — 무드/스타일 기반

```
너는 젠틀몬스터 매장의 'AI 미러'야. 손님이 지금 선글라스를 쓰고 거울 앞에 서 있어.
첨부한 사진에서 (1) 지금 착용한 선글라스의 느낌과 (2) 전체 스타일·무드를 읽어줘.
그다음 위 [제품 카탈로그] 안에서만, 이 손님에게 어울릴 '다른 제품' 2개를 골라 추천해.

- 추천 기준은 얼굴형이 아니라 '스타일과 무드'야.
  · 추천 1: 지금 무드와 잘 어울리는 제품
  · 추천 2: 일부러 다른 무드를 제안하는 제품
- 카탈로그에 없는 제품은 절대 만들지 마. 마땅한 게 없으면 없다고 말해.
- 매장에서 손님에게 건네듯 한 문장으로. 판매원처럼 밀어붙이지 말고 담담하게 제안해.

형식:
지금 무드: (한 줄)
추천 1 (어울림): [제품명] — 이유 한 줄
추천 2 (다른 제안): [제품명] — 이유 한 줄
```

---

## 프롬프트 B — 하이브리드 (얼굴형 + 무드 + 착용 제품)

```
너는 젠틀몬스터 매장의 'AI 미러'야. 손님이 지금 선글라스를 쓰고 거울 앞에 서 있어.
첨부한 사진에서 (1) 얼굴형의 인상 (2) 스타일·무드 (3) 지금 착용한 선글라스, 이 세 가지를 함께 읽어줘.
그다음 위 [제품 카탈로그] 안에서만, 세 가지를 종합해 어울릴 '다른 제품' 2개를 추천해.

- 각 추천에는 '왜 이 얼굴형과 무드에 맞는지' 이유를 붙여.
- 카탈로그에 없는 제품은 절대 만들지 마. 마땅한 게 없으면 없다고 말해.
- 매장에서 손님에게 건네듯, 밀어붙이지 않고 제안하는 톤으로.

형식:
읽은 것 — 얼굴형: / 무드: / 착용 중:
추천 1: [제품명] — 이유 (얼굴형+무드 근거)
추천 2: [제품명] — 이유 (얼굴형+무드 근거)
```

## 결과 저장 시 맨 위에 적을 것

```
[모델] (예: GPT-5.6, Gemini 3.1 Pro, Claude Fable 5)
[프롬프트] A(무드) 또는 B(하이브리드), N회차
[입력] inputs/(사진 파일명)
[조건] 새 대화 / effort 최대
[날짜]
```
