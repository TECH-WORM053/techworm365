## 0. 공고 분석 — 이 플랜의 기준점

공고에서 포트폴리오에 대해 명시한 조건:

- "단순 AI 생성 이미지 결과물이 아닌, AI를 다양한 방식으로 탐구하고 확장한 **과정**이 드러나야 합니다."
- 아래 중 1개 이상 포함 (이 플랜은 **4개 전부** 충족하도록 설계):
  1. 서로 다른 유형의 AI 도구를 활용한 실험 (생성, 자동화, 인터랙션 등)
  2. 하나의 기술을 다양한 방식으로 적용해본 탐구 과정
  3. AI를 활용해 새로운 경험, 서비스, 콘텐츠로 확장해본 시도
  4. 다양한 AI 도구를 비교하고 실제 활용 가능성을 실험한 기록
- "완성도보다 시도, 탐구 과정, 사고 방식을 중요하게 봅니다."

담당업무 중 "AI 관련 인사이트를 아카이빙하고 공유합니다"가 있으므로,
**실험 기록(아카이브) 자체가 곧 직무 능력 증명**이 됩니다.

## 1. 구현 수준 — 추천 순위

질문 주셨던 "어느 수준까지 만들 것인가"에 대한 순위입니다.

**결론: 6개월이 있으므로 셋 중 하나를 고르는 게 아니라, 세 가지를 단계(Phase)로 쌓는 것을 추천합니다.**
컨셉 기획이 1층, 도구 실험이 2층, 프로토타입이 3층인 구조입니다.

만약 시간이 부족해져서 우선순위를 정해야 한다면:

| 순위 | 수준 | 이유 |
|---|---|---|
| 1위 | **간단한 인터랙션 프로토타입** | 공고의 실험 유형에 "인터랙션"이 명시되어 있고, 지원자 대부분이 이미지·영상 결과물만 낼 가능성이 높아 작동하는 데모 하나가 확실한 차별점이 됩니다. 웹캠 + AI 파이프라인 수준이면 충분하고, 코딩은 Claude와 함께 진행합니다. |
| 2위 | **AI 도구 실험·비교 기록** | 공고의 4가지 포트폴리오 요건 중 3개를 이것만으로 충족합니다. 프로토타입이 실패해도 이 기록만으로 포트폴리오가 성립합니다. 사실상 '보험'이자 본체입니다. |
| 3위 | **컨셉 기획 + 비주얼 시나리오** | 필요하지만 이것만 단독으로는 "단순 AI 생성 이미지 결과물"로 보일 위험이 있습니다. 1·2위의 뼈대 역할로 씁니다. |

## 2. 문제 정의 (Problem Statement)

**관찰**
젠틀몬스터 매장은 공간 연출과 오브제로 '낯선 감각'을 만드는 데 집중해 왔지만,
정작 고객이 브랜드와 가장 밀접하게 만나는 순간 — **선글라스를 쓰고 거울을 보는 순간** — 은
여전히 가장 기능적이고, 어느 안경점과도 다르지 않은 경험으로 남아 있다.

**문제 정의 (한 문장)**
> 구매 결정의 감정적 정점인 '거울 앞의 순간'이, 브랜드 경험에서 가장 비어 있는 순간이다.
> AI는 이 순간을 제품 확인이 아닌, 개인화된 브랜드 경험으로 바꿀 수 있는가?

**왜 지금인가**
- 비전 AI가 사진 한 장에서 얼굴형·스타일·분위기를 해석할 수 있는 수준에 도달
- 이미지·영상 생성 속도가 '매장에서 기다릴 수 있는 시간' 안으로 들어오기 시작
- 리테일 개인화는 대부분 '추천'에 머물러 있고, '감각적 경험'으로서의 개인화는 공백

## 3. 핵심 리서치 질문 (Research Questions)

프로젝트 전체를 5개의 질문으로 끌고 갑니다. 각 질문이 하나의 실험과 연결됩니다.

- **RQ1. 해석** — 고객이 선글라스를 쓴 순간, AI는 무엇을 '읽을' 수 있는가?
  얼굴형·스타일·감정 무드·착용 제품·공간 분위기 중 어디까지가 의미 있게 해석 가능한가?
- **RQ2. 변환** — 읽어낸 것을 어떤 형태(이미지·영상·문장·사운드)로 되돌려줄 때
  사람은 "이건 나를 위한 것"이라고 느끼는가?
- **RQ3. 경계** — 초현실적 연출은 어디까지가 '젠틀몬스터답고', 어디부터 이질적인가?
  얼굴을 그대로 쓸 때와 추상화할 때, 불쾌함(언캐니)의 경계는 어디인가?
- **RQ4. 실현 가능성** — 매장이라는 환경에서 실제로 작동하려면?
  생성 대기 시간, 품질, 조명·혼잡도 같은 현실 변수의 한계는 어디인가?
- **RQ5. 신뢰** — 얼굴 데이터를 쓰는 경험에서 고객이 불편함 없이 참여하게 하는 조건은 무엇인가?

## 4. 사용자 경험 시나리오 (Experience Scenario)

### 페르소나
- **A. 첫 방문 외국인 관광객** — 브랜드를 SNS로만 알고 성수 플래그십에 처음 방문
- **B. 재방문 로컬 고객** — 이미 한 개 보유, 신제품을 구경하러 옴

### 경험 여정 (5단계)

| 단계 | 이름 | 경험 내용 |
|---|---|---|
| 1 | 착용 (Wear) | 고객이 매대에서 선글라스를 집어 쓰고 AI Mirror 앞에 선다. 거울은 평범한 거울처럼 보인다. |
| 2 | 인식 (Read) | 거울이 조용히 반응하기 시작한다. 착용 제품, 얼굴형, 스타일, 표정 무드, 매장 공간의 톤을 읽는다. |
| 3 | 해석 (Interpret) | 화면 가장자리에 해석이 시(詩)적인 짧은 문장으로 나타난다. 예: "오늘의 당신은 차분한 직선 — MM004의 각과 닮았습니다." |
| 4 | 변환 (Transform) | 거울 속 배경이 서서히 초현실 장면으로 바뀐다. 고객의 실루엣과 착용 제품은 유지된 채, 무드에 맞는 젠틀몬스터적 세계(예: 금속 사막, 부유하는 조각)로 이동한다. |
| 5 | 확장 (Take-out) | QR 코드로 '나의 무드 필름'(4~6초 영상)을 받아간다. 제품명·매장 위치가 자연스럽게 담겨 SNS 공유가 곧 브랜드 콘텐츠가 된다. |

핵심 설계 원칙: **거울은 판매하지 않는다. 해석해 줄 뿐이다.**
(추천·구매 유도가 아니라 '나를 읽어주는 경험' 자체가 목적 — 이것이 브랜드 경험과 커머스의 차이)

## 5. AI 활용 방식 — 도구 유형별 역할

공고의 "서로 다른 유형의 AI 도구" 요건에 맞춰 4가지 유형을 모두 사용합니다.

| 유형 | 역할 | 후보 도구 (실험으로 비교·확정) |
|---|---|---|
| 해석 (Vision-LLM) | 착용 사진에서 얼굴형·스타일·무드·공간 읽기 | GPT-4o/5, Claude, Gemini 비전 기능 |
| 생성 — 이미지 | 무드 기반 초현실 배경·장면 생성 | Midjourney, 나노바나나(Gemini 이미지), Stable Diffusion/FLUX |
| 생성 — 영상 | 무드 필름(테이크아웃 콘텐츠) 제작 | Kling, Runway, Sora 등 이미지-투-비디오 |
| 언어 (LLM) | 해석을 브랜드 톤의 짧은 문장으로 변환 | Claude/GPT + 프롬프트 설계 |
| 인터랙션 (코드) | 웹캠 → 해석 → 생성 → 표시 파이프라인 | 웹 기반 프로토타입 (Claude Code와 함께 제작) |
| 자동화 | 실험 기록·이미지 정리 파이프라인 | 스크립트로 실험 로그 자동 아카이빙 |

## 6. 실험 과정 (Experiment Log 설계) — 포트폴리오의 본체

각 실험은 동일한 기록 형식을 따릅니다:
**질문 → 방법 → 결과(스크린샷·표) → 배운 것 → 다음 실험으로 이어지는 결정**

| # | 실험 이름 | 연결 RQ | 내용 | 공고 요건 |
|---|---|---|---|---|
| 1 | 거울은 무엇을 읽는가 | RQ1 | 같은 착용 사진 세트를 3개 비전 모델에 주고 얼굴형·무드·스타일 해석을 비교. 어떤 모델이 '시적 해석'에 강한지, 어디까지 일관적인지 기록 | 도구 비교 ④ |
| 2 | 해석을 장면으로 | RQ2 | 실험 1의 해석 결과를 프롬프트로 변환해 이미지 생성 도구 3종 비교. '젠틀몬스터적 초현실'을 가장 잘 만드는 도구·프롬프트 구조 탐색 | 도구 비교 ④, 생성 실험 ① |
| 3 | 나인가, 낯선 나인가 | RQ3 | 개인화 강도 실험: 실제 얼굴 유지 / 실루엣만 / 완전 추상화 3단계를 만들어 주변인 5~10명에게 보여주고 반응 기록 (언캐니 경계 찾기) | 한 기술의 다양한 적용 ② |
| 4 | 움직이는 무드 | RQ2 | 같은 이미지를 여러 영상 생성 도구로 무드 필름화. 4~6초 안에서 '브랜드 필름' 느낌이 나는 조건 탐색 | 도구 비교 ④ |
| 5 | 브랜드의 목소리 | RQ2·3 | LLM 프롬프트 설계 실험: 젠틀몬스터 톤의 해석 문장 만들기. 톤 가이드를 프롬프트에 넣는 방식 여러 개 비교 | 한 기술의 다양한 적용 ② |
| 6 | 거울을 작동시키다 | RQ4 | 웹캠 프로토타입 제작: 사진 촬영 → 비전 해석 → 문장 + 이미지 생성 → 화면 표시. 대기 시간 측정, 매장 조건(조명 등) 시뮬레이션 | 인터랙션 ①, 경험 확장 ③ |
| 7 | 신뢰의 조건 | RQ5 | 얼굴 데이터 처리 방식 정리(즉시 삭제, 옵트인 등) + 프로토타입에 동의 화면 반영 | 경험 확장 ③ |

**운영 원칙**
- 실패한 실험도 기록한다. 오히려 "왜 안 됐는지"가 탐구 과정의 증거.
- 실험마다 1~2장의 비교 이미지/표 + 300자 내외의 배운 점. 분량보다 사고의 흐름.
- 실험 기록은 진행하면서 바로바로 남긴다 (마지막에 몰아서 재구성하지 않기).

## 9. 젠틀몬스터와의 연결성

- **팀 미션과의 직접 일치**: 공고의 팀 소개 "AI가 만들어낼 수 있는 낯선 감각과 가능성을 발견하고, 이를 브랜드의 리테일·콘텐츠·공간 경험으로 확장" — 이 프로젝트는 그 문장을 그대로 실행한 사례가 되도록 설계.
- **브랜드 문법 계승**: 젠틀몬스터 매장은 이미 키네틱 오브제 등 '기계가 만드는 낯섦'을 공간의 언어로 써 왔음. AI Mirror는 그 언어를 '고객 개인'에게 적용하는 다음 단계라는 논리로 연결.
- **커머스가 아닌 경험**: 추천 엔진이 아니라 해석 경험이라는 점이, 기술을 기능으로 보지 않는다는 팀 철학과 맞닿음.
- **콘텐츠 순환**: 테이크아웃 무드 필름이 SNS 공유 → 브랜드 콘텐츠로 순환되는 구조는 리테일 경험과 콘텐츠를 하나로 묶음 (담당업무의 "리테일·공간·콘텐츠 연결").

## 10. 포트폴리오 웹페이지 구성 (사이트맵)

한 페이지 스크롤형 (젠틀몬스터 웹사이트의 미니멀한 톤 참고):

1. **Hero** — 프로젝트 한 줄 정의 + 대표 비주얼(프로토타입 작동 GIF 또는 컨셉 이미지)
2. **Why** — 문제 정의 (거울 앞의 비어 있는 순간)
3. **Questions** — 5개 리서치 질문
4. **Experience** — 5단계 경험 여정 (시나리오 + 컨셉 비주얼)
5. **Experiments** — 실험 1~7 아카이브. **분량과 위치 모두 페이지의 중심.** 실패 기록 포함
6. **Prototype** — 데모 영상 + 파이프라인 구조도(그림 1장)
7. **Findings** — 실험을 관통하는 인사이트 3~5개 (예: "개인화는 정확할수록 좋은 게 아니라, 시적일수록 좋았다")
8. **What's Next** — 매장 적용 시 필요한 것, 다음 실험 계획 (탐구가 끝나지 않았음을 보여주기)
9. **About** — 짧은 소개 + 사용 도구 목록 + 공개 실험 채널 링크

플랫폼 (확정): **Claude Code로 직접 코딩한 웹페이지**
- 이미 Claude Code로 랜딩페이지를 만들어본 경험이 있어 실현 가능성 확인됨
- 자유도가 가장 높아 젠틀몬스터 감성의 미니멀·모션 연출 가능
- "포트폴리오 페이지 자체도 AI와 함께 만들었다"는 사실이 곧 AI 활용 증거가 됨
  → About 섹션에 제작 과정(Claude Code 활용 방식)을 한 단락으로 기록할 것
- 예비안: 일정이 밀리면 Notion으로 우선 제출하고 이후 교체
