# Living Window — 도시에 반응하는 쇼윈도 (프로젝트 3 플랜 v1)

> 포지셔닝: 서브 프로젝트. 메인(AI Mirror) 프로토타입 완료 후 진행하는 **4주 스프린트** (5~6개월차)
> 작성일: 2026-07-03
> 다른 프로젝트: AI Mirror Experience(메인, **개인**에 반응) · LENS(프로젝트 2, 흐름을 읽기)
> 축 구분: Mirror가 '한 사람'을 읽는다면, Living Window는 '도시의 조건(날씨·시간·계절)'을 읽는다 — 같은 질문("AI는 무엇을 읽고 무엇으로 되돌려주는가")의 다른 스케일

---

## 1. 문제 정의

**관찰**
젠틀몬스터 매장은 공간 연출로 유명하지만, 그 공간은 **시즌 단위로 바뀌는 '설치'** 다.
시즌 안에서 공간은 정지해 있다. 오늘 아침과 오늘 밤, 맑은 날과 비 오는 날의 쇼윈도가 똑같다.
그리고 쇼윈도는 매장에 들어오지 않는 사람까지 포함해 **브랜드가 가장 많은 사람과 만나는 접점**인데,
정작 가장 정적인 매체로 남아 있다.

**문제 정의 (한 문장)**
> 이미지·영상 생성 비용이 급격히 떨어진 지금,
> 공간 연출은 '고정된 작품'에서 '도시의 리듬에 반응하는 생명'으로 바뀔 수 있는가?

**왜 지금인가**
- 생성 AI로 하나의 공간 세계관을 수십 가지 조건(날씨×시간)으로 변주하는 비용이 사실상 0에 수렴
- 날씨·시간·계절 데이터는 공개 API로 실시간 확보 가능 — 기술 장벽이 아니라 상상력의 문제
- 행인을 촬영하지 않고 환경 데이터만 쓰므로, Mirror와 달리 프라이버시 부담이 없는 접근

---

## 2. 핵심 리서치 질문

- **RQ1. 입력** — 공간이 무엇에 반응할 때 의미가 있는가? (날씨·시간대·계절·소음·유동인구 중 어떤 조건이 '반응할 가치'가 있나)
- **RQ2. 리듬** — 어느 속도와 강도로 변할 때 '살아 있다'고 느껴지는가? (너무 빠르면 전광판, 너무 느리면 아무도 모른다)
- **RQ3. 일관성** — 수십 개의 생성 변주가 하나의 브랜드 세계로 보이려면 무엇을 고정해야 하는가?
- **RQ4. 물성** — 화면 속 생성 이미지가 젠틀몬스터의 물리적 오브제(키네틱) 전통과 만나는 지점은 어디인가?

## 3. 사용자 경험 시나리오

'사용자'는 매장 앞을 지나가는 행인 — 고객이 되기 전의 사람들입니다.

### 하나의 쇼윈도, 세 개의 순간 (가상 테마: "금속 정원")

| 장면 | 조건 | 쇼윈도의 상태 |
|---|---|---|
| A. 맑은 오후 | 맑음, 14시 | 금속 식물이 빛을 받아 천천히 피어 있다. 그림자가 실제 태양 방향과 일치한다 |
| B. 비 오는 밤 | 비, 21시 | 같은 정원이 젖은 금속의 세계로 바뀌어 있다. 낮에 본 사람이 밤에 다시 보면 '다른 장면'이 아니라 '같은 세계의 다른 시간'을 느낀다 |
| C. 새벽 | 눈, 5시 | 아무도 없는 거리, 정원은 얼어붙은 채 아주 느리게 숨 쉰다. 우연히 본 한 사람만을 위한 장면 |

### 행인의 여정 (4단계)

1. **스침** — 퇴근길, 어제도 지나친 쇼윈도를 지나간다
2. **위화감** — "어제랑… 다른데?" 걸음이 느려진다 (변화를 '알아채는' 순간이 이 경험의 핵심)
3. **확인** — 멈춰 서서 본다. 비가 오는 지금, 쇼윈도 안에도 비의 세계가 있다
4. **관계** — 다음부터 이 앞을 지날 때마다 확인하게 된다. 쇼윈도가 '전시'에서 '안부를 확인하는 존재'로 바뀐다

핵심 설계 원칙: **광고판처럼 눈길을 끌지 않고, 생명처럼 알아차리게 한다.**

## 4. AI 활용 방식 — 도구 유형별 역할

| 유형 | 역할 | 비고 |
|---|---|---|
| LLM | 테마의 세계관 설계 + 조건별 장면 묘사문 생성 (조건→무드 번역기) | Mirror 실험 5(브랜드 톤)의 성과 재활용 |
| 이미지 생성 | 하나의 테마를 날씨×시간 조건으로 수십 장 변주 | 도구 비교는 Mirror 실험 2 결과에서 출발 — 중복 실험 없이 '공간 스케일 표현' 검증만 추가 |
| 영상 생성 | 장면 사이의 전환(모핑)과 미세한 루프 움직임 | Mirror 실험 4 성과 재활용 |
| 인터랙션 (코드) | 실제 날씨 API + 시각 → 장면 자동 선택·전환하는 시뮬레이터 | Claude Code로 제작, 모니터 1대로 시연 |
| 자동화 | 조건 조합별 이미지 일괄 생성·정리 파이프라인 | LENS에서 만든 자동화 경험 재활용 |

세 프로젝트의 도구 경험이 여기서 합류하도록 설계 — "실험이 실험으로 이어진다"는 서사를 포트폴리오 전체에 부여.

## 5. 실험 과정 (4주 스프린트)

기록 형식은 공용 실험로그 템플릿(templates/실험로그-템플릿.md)을 따릅니다.

| # | 실험 이름 | 연결 RQ | 내용 | 공고 요건 |
|---|---|---|---|---|
| 1 | 하나의 세계, 스무 개의 순간 | RQ3 | 테마 1개를 날씨 5종 × 시간대 4종 = 20개 조건으로 변주 생성. 스타일 고정 기법(레퍼런스 이미지, 스타일 프롬프트, 시드 고정 등)을 비교해 '같은 세계'로 보이는 조건 찾기 | 한 기술의 다양한 적용 ② |
| 2 | 공간을 그리는 도구 | RQ3 | 공간·건축 스케일 표현력 기준으로 이미지 도구 2~3종 재비교 (Mirror 실험 2의 후속 — 인물 기준과 공간 기준의 순위가 달라지는지) | 도구 비교 ④ |
| 3 | 살아있음의 최소 속도 | RQ2 | 같은 장면을 ①정지 이미지 교체 ②느린 모핑 전환 ③미세 루프 영상으로 만들어 주변인에게 노출. "언제 변화를 알아챘는지 / 광고판 같은지 생명 같은지" 반응 기록 | 한 기술의 다양한 적용 ② |
| 4 | 창문을 켜다 | RQ1·2 | 실제 날씨 API + 시간 연동 시뮬레이터 제작. 모니터를 창가에 1~2일 두고 실제 날씨 변화에 반응하는 모습 타임랩스 기록 | 인터랙션 ①, 경험 확장 ③ |
| 5 | (선택) 화면 너머 | RQ4 | 프로젝션·키네틱 오브제와의 결합 컨셉 스케치 — 물리 제작 없이 생성 이미지로 아이디어만 시각화 | 경험 확장 ③ |

**운영 원칙**
- 4주 타임박스. 실험 5는 시간이 남을 때만 (없어도 프로젝트 성립)
- Mirror에서 검증된 것은 재실험하지 않고 결과를 인용 — "이전 실험이 다음 실험의 출발점"임을 로그에 명시

## 6. 최종 산출물

1. **조건 반응 시뮬레이터** — 실제 날씨 API 연동 웹 데모 (포트폴리오에서 직접 작동)
2. **타임랩스 영상 (30초~1분)** — 실제 날씨가 바뀌는 하루 동안 시뮬레이터가 반응하는 기록
3. **변주 아카이브** — 20개 조건 이미지를 한눈에 보는 그리드 (일관성의 증거)
4. **실험 로그 4~5건**
5. **컨셉 필름 (선택)** — 실제 매장 쇼윈도에 설치된 모습의 생성 영상 연출

## 7. 평가 기준

| 기준 | 측정 방법 |
|---|---|
| 세계관 일관성 | 변주 20장을 무작위로 섞고 다른 브랜드 무드 이미지와 혼합 → 주변인이 '같은 세계'를 골라내는 정확도 |
| 반응 인지 | 실험 3에서 변화를 알아챈 시점과 첫 반응 ("광고판 같다" vs "살아있는 것 같다" 분류) |
| 실현 가능성 | 조건 전환 시 생성·전환 지연 시간, 이미지 해상도의 대형 화면 확대 한계 실측 |
| 브랜드 적합성 | Mirror와 동일한 방법 — 실제 젠틀몬스터 공간 사진과 섞어 이질감 평가 |
| 관계 형성 가능성 | 정성 평가: "매일 지나간다면 다시 보고 싶은가" 질문 |

## 8. 젠틀몬스터와의 연결성

- **공간이라는 본진**: 젠틀몬스터의 브랜드 정체성이 가장 강한 영역이 공간 연출. 담당업무의 "AI 기술을 리테일, **공간**, 콘텐츠 등 브랜드 경험에 연결"에서 '공간'을 정면으로 다루는 프로젝트.
- **키네틱 전통의 계승**: 젠틀몬스터는 이미 움직이는 기계 오브제로 '살아있는 공간'을 만들어 온 브랜드. Living Window는 그 언어를 '조건에 반응하는 생성 비주얼'로 확장하는 다음 단계라는 논리.
- **포트폴리오 전체의 뼈대 완성**: Mirror(사람에 반응) → Living Window(환경에 반응) → LENS(흐름을 읽기). 셋이 모여 "AI는 무엇을 읽고, 무엇으로 되돌려주는가"라는 하나의 질문을 리테일·공간·콘텐츠 세 영역에서 탐구한 포트폴리오가 됨.
- **매장 밖까지 보는 시야**: 구매 고객만이 아니라 행인 — 아직 고객이 아닌 사람 — 과의 접점을 다룬다는 점에서 브랜드 경험의 정의를 넓히는 제안.

## 9. 포트폴리오 페이지 구성

포트폴리오 사이트 내 별도 섹션 (Mirror, LENS 다음):

1. **Hero** — 타임랩스 영상 (실제 날씨에 반응하는 시뮬레이터)
2. **Why** — 문제 정의: 가장 많은 사람과 만나는 가장 정적인 매체
3. **Three Moments** — 맑은 오후/비 오는 밤/새벽, 세 장면 시나리오
4. **Experiments** — 실험 로그 (특히 '같은 세계' 그리드와 '살아있음의 최소 속도' 반응 기록)
5. **Try it** — 작동하는 시뮬레이터 임베드 (보는 사람의 현재 날씨에 반응)
6. **Beyond the Screen** — 키네틱 결합 스케치 + 다음 질문들

## 10. 일정 — 전체 로드맵 속 위치

| 기간 | 할 일 |
|---|---|
| 5개월차 1~2주 | 실험 1·2 (변주 생성, 도구 재비교) — Mirror 포트폴리오 페이지 작업과 병행 |
| 5개월차 3~4주 | 실험 3 (전환 속도 테스트) + 시뮬레이터 제작 착수 |
| 6개월차 1~2주 | 실험 4 (시뮬레이터 완성, 타임랩스 기록), 선택 실험 5 |
| 6개월차 3~4주 | 페이지 섹션 제작·통합, 전체 포트폴리오 마무리 |

전제: Mirror 프로토타입(4개월차)이 끝난 뒤 시작 — 도구 비교·톤 설계·코딩 파이프라인 경험을 그대로 물려받아 **절반의 노력으로 두 번째 프로젝트를 완성**하는 구조.

## 11. 리스크와 대비

| 리스크 | 대비 |
|---|---|
| Mirror와 비슷해 보일 위험 | 축을 명시적으로 구분: Mirror는 개인(얼굴·무드), Living Window는 환경(날씨·시간). 페이지에도 이 대비를 한 문장으로 박아둠 |
| 일정 밀림 (메인이 늦어질 때) | 4주 스프린트 자체를 축소 가능하게 설계: 실험 1+3+시뮬레이터만으로도 성립 (실험 2·5는 선택) |
| 대형 화면 품질 검증 불가 | 실제 쇼윈도 크기 검증은 불가능함을 솔직히 명시하고, 모니터/TV 수준 실측 + 업스케일 실험으로 한계와 함께 기록 — '한계를 아는 것'도 리서치 결과 |
| 시뮬레이터 기술 난이도 | Mirror 프로토타입보다 단순 (카메라·실시간 해석 없음, 날씨 API + 이미지 전환뿐). Mirror를 먼저 하는 이유 |
| 생성 변주의 일관성 실패 | 실패해도 "무엇을 고정해도 세계가 흩어지는가"가 실험 1의 정당한 결론이 됨 — 기록으로 전환 |
