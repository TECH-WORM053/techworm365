# TouchDesigner 포인트클라우드 모핑 — 초보자용 영상 분석 설명서

분석 영상: [morphing particles and pointclouds in touchdesigner using TOPs](https://www.youtube.com/watch?v=sE716KGZw0w) — wxll, 27분 51초

## 1. 이 영상이 만드는 것

완성 데모는 **MediaPipe로 모핑을 제어**하지만, 튜토리얼 본편에서는 구조를 쉽게 설명하기 위해 MediaPipe 대신 키보드 `1`을 사용한다. 두 개의 `.ply` 포인트클라우드 파일을 TouchDesigner로 불러온 뒤, 수많은 점으로 렌더링하고 키를 눌렀다 놓으면 점들이 흩어지면서 용과 나비 사이를 바꾸는 작업이다.

가장 쉽게 비유하면 다음과 같다.

- `pos`는 모든 점의 **주소록**이다. 각 픽셀의 R/G/B 값이 점 하나의 X/Y/Z 위치가 된다.
- `col`은 모든 점의 **색상표**다. 각 픽셀의 R/G/B 값이 같은 번호 점의 색상이 된다.
- `Switch TOP`은 용 주소록과 나비 주소록 사이를 바꾼다.
- `Noise TOP`은 각 점의 주소에 임시 흔들림을 더해 점들을 흩뜨린다.
- `Geometry COMP`는 주소록의 픽셀 수만큼 작은 점을 GPU로 한꺼번에 그린다.

즉, 완성 구조는 아래와 같다.

```text
용 PLY ── 위치 ── pos_dragon ─┐
                              ├─ switch_pos ─ noise와 합산 ─ POS ─┐
나비 PLY ─ 위치 ─ pos_butterfly┘                                  │
                                                                  ├─ geo_particles ─ render1
용 PLY ── 색상 ── col_dragon ─┐                                   │
                              ├─ switch_col ─────────────── COL ──┘
나비 PLY ─ 색상 ─ col_butterfly┘

키보드 1 ── AMP   ── Noise 세기
          └ INDEX ── 두 Switch의 Index
```

## 2. 시작 전에 준비할 것

1. TouchDesigner의 새 빈 프로젝트
2. 점과 색상 정보가 들어 있는 `.ply` 파일 2개
3. 가능하면 두 파일의 점 개수가 비슷한 것

영상에 사용된 정확한 용·나비 PLY는 영상 설명란의 Patreon 자료다. 그 파일이 없어도 다른 PLY 두 개로 구조는 그대로 연습할 수 있다. 일반적인 FBX 메시를 이 네트워크에 바로 넣는 방식은 아니다.

TouchDesigner의 `Point File In TOP`은 PLY의 점 데이터를 불러와 위치 등의 값 네 개를 RGBA 텍스처에 담는다. 추가 속성은 `Point File Select TOP`으로 별도 텍스처에 꺼낼 수 있다.

### 초보자가 알아야 할 노드 색상

- 보라색 `TOP`: 이미지와 GPU 데이터. 이 영상의 핵심이다.
- 초록색 `CHOP`: 키보드 값과 애니메이션 숫자.
- 회색 `COMP`: 카메라, Geometry, Light 같은 3D 오브젝트.
- 노란색 `MAT`: 점이나 선의 재질.
- 파란색 `SOP`: Box, Sphere, Add 같은 3D 모양.

## 3. 1단계 — 첫 번째 PLY의 위치 만들기

빈 공간에서 `Tab`을 누르고 다음 노드를 만든다.

```text
Point File In TOP → Point Transform TOP → Fit TOP → Fit TOP → Null TOP
```

이름을 아래처럼 바꾼다.

```text
dragon_file → dragon_transform → dragon_pos_2x → dragon_pos_limit → pos_dragon
```

설정:

1. `dragon_file`의 **File**에서 첫 번째 `.ply`를 선택한다.
2. `dragon_transform`은 모델의 중심·회전·크기를 맞추는 용도다. 처음에는 기본값으로 둔다.
3. 첫 Fit의 `Common → Output Resolution`을 `2X`로 둔다.
4. 두 번째 Fit은 `Limit Resolution`으로 바꾸고 `800 × 800`으로 제한한다.
5. 두 Fit 모두 Input Smoothness와 Viewer Smoothness를 `Nearest Pixel`로 둔다.
6. `pos_dragon`은 마지막 확인용 Null이다.

영상은 먼저 원본을 `2X`로 키운 다음, 두 번째 Fit에서 최종 크기를 `800×800`, 즉 64만 점으로 통일한다. 컴퓨터가 느리면 최종 크기를 512×512 또는 그 이하로 낮춘다.

중요한 것은 숫자 자체가 아니라 **두 모델의 위치와 색상 TOP 네 개가 전부 같은 해상도**여야 한다는 점이다.

## 4. 2단계 — 첫 번째 PLY의 색상 만들기

`dragon_file`에서 색상용 가지를 하나 더 만든다.

```text
dragon_file → Point File Select TOP → Math TOP → Fit TOP → Fit TOP → Null TOP
```

이름:

```text
dragon_color_select → dragon_color_math → dragon_col_2x → dragon_col_limit → col_dragon
```

설정:

1. `dragon_color_select`에서 PLY의 색상 필드가 R/G/B로 나오게 한다.
2. 색상 필드 이름은 파일마다 `red/green/blue`, `r/g/b`, `Cd`처럼 다를 수 있다.
3. 영상의 PLY에서는 `Math TOP`의 Multiply 값을 사용해 색상 값을 **127로 나눈다**.
4. 위치와 똑같이 Fit 두 개를 사용해 `2X → 800×800`으로 만든다.
5. 두 Fit의 Smoothness를 `Nearest Pixel`로 둔다.

색상 노드 화면이 실제 사진처럼 보이지 않고 붉거나 잡음처럼 보여도 정상이다. 이 TOP은 감상용 사진이 아니라 점마다 사용할 숫자표다.

## 5. 3단계 — 두 번째 PLY도 똑같이 만들기

위의 위치·색상 가지를 복제해 두 번째 파일에 연결한다.

```text
butterfly_file → butterfly_transform → Fit(2X) → Fit(800×800) → pos_butterfly

butterfly_file → butterfly_color_select → butterfly_color_math
               → Fit(2X) → Fit(800×800) → col_butterfly
```

완료 후 다음 네 노드를 가운데 버튼으로 눌러 해상도를 확인한다.

- `pos_dragon`
- `pos_butterfly`
- `col_dragon`
- `col_butterfly`

영상 기준으로 네 개가 모두 `800 x 800`, 즉 640,000픽셀이어야 한다. 이 조건이 맞지 않으면 전환할 때 점이 깨지거나 Switch가 정상적으로 섞이지 않는다.

`Point Transform TOP`에서 두 모델의 중심과 크기도 비슷하게 맞춘다. 한 모델만 지나치게 크거나 중심에서 벗어나 있으면 모핑 중 화면 밖으로 날아간 것처럼 보인다.

## 6. 4단계 — 위치와 색상 Switch 만들기

`Switch TOP` 두 개와 `Null TOP` 두 개를 만든다.

```text
pos_dragon ─────┐
                ├→ switch_pos → (나중에 Noise 추가) → POS
pos_butterfly ──┘

col_dragon ─────┐
                ├→ switch_col → COL
col_butterfly ──┘
```

두 Switch의 입력 순서는 반드시 동일해야 한다.

- 입력 0: 용 위치 / 용 색상
- 입력 1: 나비 위치 / 나비 색상

두 Switch에서 `Blend between Inputs`를 **On**으로 켠다. 그러면 Index가 0.5일 때 두 입력의 중간값을 만들 수 있고, 모든 점이 동시에 부드럽게 모핑된다.

먼저 Index를 손으로 0과 1 사이에서 움직여 본다. `switch_pos`와 `switch_col`이 같은 모델을 가리키는지 확인한다.

## 7. 5단계 — 점으로 렌더링하기

다음 노드를 만든다.

```text
Geometry COMP   : geo_particles
Camera COMP     : cam1
Light COMP      : light1
Render TOP      : render1
Point Sprite MAT: pointsprite1
```

### geo_particles 안쪽

1. `geo_particles` 안으로 들어간다.
2. 기본 `torus1`은 지운다.
3. `Add SOP`을 만들고 `Add Points`를 켠다.
4. 뒤에 `Convert SOP`을 연결한다.
5. Convert To를 `Particles per Point`, Particle Type을 `Render as Point Sprites`로 설정한다.
6. Convert SOP의 보라색 Render 플래그와 파란색 Display 플래그를 켠다.
7. 다시 `/project1`로 나온다.

점 하나가 없으면 수백만 번 인스턴싱해도 그릴 원본이 없어 화면이 검게 나온다.

### 위치 인스턴싱

`geo_particles` 파라미터의 `Instance` 페이지에서:

- `Instancing`: On
- `Instance Count Mode`: Instance OP(s) Length
- `Translate OP`: `POS`
- `Translate X`: `r`
- `Translate Y`: `g`
- `Translate Z`: `b`

이 설정은 POS 텍스처의 모든 픽셀을 읽어 픽셀 하나당 점 하나를 만든다.

### 색상 인스턴싱

다음 Instance 페이지에서 색상 항목을 찾는다.

- `Color OP`: `COL`
- `Color R`: `r`
- `Color G`: `g`
- `Color B`: `b`
- `Color A`: `a` 또는 비워 두기

TouchDesigner 빌드에 따라 색상 항목이 `Instance 2` 또는 그 옆 탭에 보일 수 있다.

### 재질과 렌더

1. `geo_particles`의 Material에 `pointsprite1`을 넣는다.
2. 영상의 `Point Sprite MAT`에서 `Constant Point Size`는 약 `0.7`이다.
3. 너무 작아 안 보이면 1~2로 올리고, 너무 뭉치면 0.2~0.7로 낮춘다.
4. `render1`의 Camera에 `cam1`, Geometry에 `geo_particles`를 넣는다.
5. 영상의 Render 해상도는 1280×720이다.

모델이 안 보이면 먼저 `cam1`의 Translate Z를 5 전후에서 조절한다. 모델 크기에 따라 값은 달라진다.

## 8. 6단계 — Noise로 점을 흩뜨리기

`Noise TOP`과 `Add TOP`을 만든다.

```text
switch_pos ────────┐
                   ├→ add_noise → Null TOP(POS)
noise1 ────────────┘
```

`noise1`의 해상도도 위치 TOP과 같아야 한다. `Add TOP`은 원래 위치 XYZ에 작은 임의 XYZ 값을 더해 점들을 흩뜨린다.

영상에서 확인되는 Noise 시작값:

- Type: `Simplex 3D (GPU)`
- Period: 약 `0.88`
- Harmonics: `2`
- Harmonic Spread: 약 `1.9`
- Harmonic Gain: 약 `0.55`
- Roughness: 약 `0.5`
- Exponent: `1`
- Monochrome: Off
- Aspect Correct: On
- 최종 Amplitude 범위: `0 ~ 0.37`

처음에는 Amplitude를 직접 0에서 0.1 정도로 천천히 올려 본다. 점들이 한 방향으로만 통째로 밀리면 Noise의 값 중심이 0이 아닌 상태다. 이때 `Math TOP`을 Noise 뒤에 넣고 범위를 대략 `-0.5 ~ +0.5`로 중심화한 후 Add에 연결한다.

## 9. 7단계 — 키보드 1로 모핑시키기

영상은 키보드 값을 두 갈래로 나눈다.

```text
                         ┌→ Math CHOP → Lag CHOP → AMP
Keyboard In CHOP (키 1) ┤
                         └→ Lag CHOP → Count CHOP → Lag CHOP → INDEX
```

### A. AMP 가지: 흩어짐 세기

1. `Keyboard In CHOP`을 만들고 `Keys`에 `1`을 쓴다.
2. `Channel Names`는 `by Key Name`으로 둔다.
3. `Math CHOP`의 Range에서 입력 `0~1`을 출력 `0~0.37`로 바꾼다.
4. 그 뒤에 `Lag CHOP`을 붙여 갑작스러운 변화를 부드럽게 한다.
5. 마지막 Null 이름을 `AMP`로 한다.
6. `AMP` 채널을 `noise1`의 Amplitude로 드래그해 `Export CHOP`을 선택한다.

추천 시작값:

- Lag 증가/감소: 영상 값 `0.7 / 0.2초`
- 최대 Amplitude: 처음에는 0.1, 잘 되면 0.37

### B. INDEX 가지: 용과 나비 교체

1. 키보드 뒤에 `Lag CHOP`을 두고 영상처럼 증가/감소를 `0.5 / 0.2초`로 시작한다.
2. `Count CHOP`은 **감소하는 값이 0.7 아래로 내려갈 때** 카운트하도록 설정한다. 즉 키를 누를 때가 아니라 놓을 때 모델이 바뀐다.
3. `Limit → Loop Min/Max`를 켜고 0~1로 반복하도록 한다.
4. 두 번째 `Lag CHOP`으로 0↔1 변화를 부드럽게 한다.
5. 마지막 Null 이름을 `INDEX`로 한다.
6. `INDEX`를 `switch_pos`와 `switch_col`의 Index에 각각 Export한다.

추천 시작값:

- 앞 Lag 증가/감소: `0.5 / 0.2초`
- 뒤 Lag: 모핑 속도를 보며 부드럽게 조절

두 Switch가 항상 같은 INDEX를 받아야 위치는 나비인데 색상은 용인 오류가 생기지 않는다.

### 기대 동작

1. 키 `1`을 누른다.
2. AMP가 올라가며 점이 흩어진다.
3. INDEX가 0에서 1 또는 1에서 0으로 바뀐다.
4. AMP가 내려가며 새 모델 모양으로 점이 모인다.

## 10. 8단계 — 흰색 바운딩 박스

영상 후반의 흰 큐브는 포인트클라우드와 별개의 Geometry다.

### 큐브 선

```text
Box SOP → geo_box
Wireframe MAT → geo_box의 Material
```

`Wireframe MAT`의 색을 흰색으로 하고 선 두께를 조절한다.

### 모서리의 흰 점

```text
Sphere SOP ─┐
            ├→ Copy SOP → geo_corners
Box SOP ────┘
```

Copy SOP의 첫 번째 입력에는 작은 Sphere, 두 번째 입력에는 Box의 점을 넣는다. Sphere가 너무 크면 Uniform Scale을 아주 작게 낮춘다.

`render1`의 Geometry에 `geo_particles geo_box geo_corners`를 함께 지정한다.

## 11. 9단계 — 잔상 효과는 마지막에

기본 모핑이 완성된 뒤에만 Feedback을 추가한다.

```text
render1 ─────────────────────┐
                            ├→ Composite TOP → OUT
Feedback TOP → Level → Blur ┘
                  ↑           │
                  └───────────┘
```

- `Feedback TOP`의 Target TOP을 최종 Composite로 지정한다.
- `Composite TOP` Operation은 Add 또는 Screen부터 시험한다.
- `Level TOP`의 Opacity를 0.90~0.98 정도로 두면 이전 프레임이 천천히 사라진다.
- Blur는 아주 약하게 사용한다.

Feedback을 먼저 만들면 원인이 포인트 렌더인지 피드백인지 구분하기 어려워진다.

## 13. 자주 막히는 문제

### 화면이 완전히 검다

- `geo_particles` 안에 Add SOP의 점 하나가 있는지 확인한다.
- `geo_particles`에 `pointsprite1`이 적용됐는지 확인한다.
- `render1`의 Camera와 Geometry 경로를 확인한다.
- 카메라 Z를 조절한다.

### 용은 보이는데 나비로 바꾸면 사라진다

- 두 위치 TOP의 해상도가 같은지 확인한다.
- `Point Transform TOP`으로 나비의 중심과 크기를 맞춘다.
- Switch 입력 0과 1이 실제로 연결됐는지 확인한다.

### 모핑할 때 형체가 찢어지거나 줄무늬가 생긴다

- 네 개의 최종 TOP 해상도가 모두 같은지 확인한다.
- Position TOP은 반드시 32-bit float RGBA를 사용한다.
- 서로 다른 점 순서는 원래 일대일 대응이 아니므로 중간 모양이 예쁘지 않을 수 있다. 영상은 Noise로 이 순간을 시각적으로 감춘다.

### 색이 이상하다

- `Point File Select TOP`이 실제 color 필드를 읽는지 확인한다.
- 영상 자료의 색상은 Math TOP에서 127로 나눈다. 다른 PLY는 저장 범위에 따라 255 등 다른 값이 필요할 수 있다.
- `Color OP`의 R/G/B 매핑을 확인한다.

### 키를 눌러도 아무 일도 없다

- Keyboard In CHOP 뷰어의 채널이 0에서 1로 변하는지 먼저 본다.
- TouchDesigner 창이 활성화되어 있어야 키 입력을 받는다.
- AMP와 INDEX의 초록색 Export 표시가 켜졌는지 확인한다.

### 너무 느리다

- 영상의 800×800은 64만 점이다.
- 느리면 512×512(약 26만 점) 또는 그 이하로 테스트한다.
- Point Size와 Feedback Blur를 낮춘다.
- Light는 Point Sprite 룩에 꼭 필요하지 않으므로 문제가 생기면 잠시 뺀다.

## 14. 초보자용 제작 순서 체크리스트

한꺼번에 만들지 말고 각 줄이 성공한 뒤 다음 줄로 넘어간다.

- [ ] PLY 하나의 위치가 Point File In TOP에 나온다.
- [ ] 같은 PLY의 색상이 Point File Select TOP에 나온다.
- [ ] PLY 두 개의 위치·색상 TOP 해상도가 모두 같다.
- [ ] Switch Index를 손으로 0/1로 바꾸면 데이터가 바뀐다.
- [ ] Geometry 인스턴싱으로 첫 모델이 점 형태로 보인다.
- [ ] Color OP를 연결해 모델 색상이 보인다.
- [ ] Noise Amplitude를 손으로 올리면 점이 흩어진다.
- [ ] Keyboard In CHOP의 값이 키 `1`에 반응한다.
- [ ] AMP가 Noise Amplitude를 움직인다.
- [ ] INDEX가 위치·색상 Switch 두 개를 동시에 움직인다.
- [ ] 마지막에만 큐브와 Feedback을 추가한다.

## 15. 지금 프로젝트와 연결할 때

이 영상 그대로의 1차 목표는 **키보드 1로 용↔나비 모핑**까지다. MediaPipe와 손 추적은 여기까지 성공한 다음, `keyboardin1` 대신 손 채널을 연결하면 된다.

예를 들면:

- 손을 펼친 정도 → `AMP`
- 핀치 순간 또는 두 손 거리의 임계값 → `Count CHOP` → `INDEX`

그러나 처음부터 MediaPipe를 섞으면 PLY, 렌더, Noise, 손 입력 중 무엇이 문제인지 알기 어려우므로 영상 구조부터 독립적으로 완성하는 것이 좋다.
