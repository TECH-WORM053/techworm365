# AI Mirror 프로토타입 v0.1

> 플랜의 **실험 6 "거울을 작동시키다"** 1단계 산출물.
> 웹캠 → 촬영 → Claude 비전 해석 → '근거 있는 시' 문장 표시까지의 최소 파이프라인.

## 무엇을 하나

1. 동의 화면 (실험 7 '신뢰의 조건' 반영 — 사진은 해석 후 즉시 폐기, 어디에도 저장 안 함)
2. 웹캠이 거울처럼 켜짐 (좌우 반전)
3. [읽기] 버튼 → 3·2·1 카운트다운 → 사진 1장 촬영
4. Claude가 사진을 보고 **근거 있는 시** 한 문장으로 해석
   (관찰한 사실 1개 = 앵커 + 무드 단어 + 화면 톤)
5. 문장이 화면에 떠오르고, 무드에 맞는 색이 배경에 스며듦
6. **해석까지 걸린 시간이 화면에 표시됨** (RQ4 대기 시간 측정용)

## 실행 방법

터미널에서 이 폴더로 이동한 뒤:

```powershell
cd "prototype\ai-mirror"
npm install        # 처음 한 번만
npm start
```

브라우저에서 **http://localhost:3737** 을 엽니다.

### 데모 모드 vs 라이브 모드

| 모드 | 조건 | 비용 |
|---|---|---|
| **데모** | API 키가 없을 때 자동 전환 (또는 아래처럼 강제 실행) | 0원 — 준비된 예시 문장을 보여줌 |
| **라이브** | API 키가 있을 때 | 사진 1장당 약 수십 원 수준 (Claude Opus 4.8 기준) |

데모 모드 강제 실행 (API 호출 없이 화면 흐름만 테스트할 때):

```powershell
$env:DEMO="1"; npm start
```

라이브 모드 설정 — Anthropic API 키를 환경변수로 넣고 실행:

```powershell
$env:ANTHROPIC_API_KEY="sk-ant-..."; npm start
```

> API 키는 서버(server.js) 안에서만 사용되고 브라우저로는 전달되지 않습니다.

### 조절할 수 있는 값 (server.js 상단)

- `MODEL` — 사용할 모델 (기본: `claude-opus-4-8`)
- `EFFORT` — 해석 깊이/속도 균형 (`low` 빠름 / `medium` 균형 / `high` 깊음)

## 개인정보 처리 (실험 7 연결)

- 사진은 브라우저 메모리 → 서버 메모리 → Anthropic API 요청 한 번에만 사용
- 디스크 저장 없음, 로그에 사진 데이터 기록 없음
- 라이브 모드에서는 Anthropic API 데이터 정책이 적용됨 (동의 화면에 명시)

## 실험 기록 (측정)

- 화면에 "해석까지 N초"가 표시됩니다.
- 브라우저 개발자 도구(F12) 콘솔에 매 회차 기록이 남습니다:
  시각 / 모드 / 전체 소요 초 / API 소요 초 / 해석 결과 전체
