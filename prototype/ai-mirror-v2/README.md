# AI Mirror v2 — 모션 트래킹 프로토타입

카메라 앞에 서면:

1. **실시간 트래킹** — 얼굴을 계속 따라가며 배경 이펙트(컬러 워시·글로우·파티클)가 움직임에 반응
2. **3초마다 해석 갱신** — 얼굴형·무드·스타일을 추정
3. 무드에 따라 **배경 팔레트가 부드럽게 전환**되고 **문구**가 바뀜
4. exp02 제품 카탈로그 기준 **추천 제품 1~3순위**가 이미지와 함께 표시

## 실행

```powershell
cd prototype/ai-mirror-v2
node serve.mjs
```

브라우저에서 `http://localhost:5180` → **거울 켜기**.
첫 실행은 얼굴 인식 모델(약 3MB)을 내려받느라 몇 초 걸립니다.

### 휴대폰에서 보기

```powershell
node serve.mjs --lan
```

- HTTP 주소로는 보안 정책상 휴대폰 실시간 카메라가 열리지 않습니다.
- `serve.mjs` 상단 주석대로 `key.pem`/`cert.pem`을 만들면 HTTPS로 자동 전환되어 실시간 카메라가 동작합니다.

### 제품 이미지

`assets/products/README.md` 참고. 공식 이미지를 로컬에 넣으면 카드에 표시되고, 없으면 일러스트로 대체됩니다. (저작권 때문에 git에는 올라가지 않습니다)

## 구조 — "AI 자리"는 비워둔 하이브리드

```
매 프레임   MediaPipe 얼굴 트래킹 ──▶ effects.js (배경·파티클, 실시간)
3초마다     analyzer.js ──▶ 얼굴형·무드·스타일 ──▶ catalog.js (추천 1~3위) + 문구 + 팔레트
```

- 지금 `analyzer.js`는 **기기 안 근사 계산**입니다: 랜드마크 비율로 얼굴형, 표정 수치로 무드, 옷 색으로 스타일을 추정합니다. API 비용 없이 전체 경험 흐름을 확인하기 위한 것입니다.
- 나중에 Claude API를 쓰기로 하면, 같은 형태의 결과를 돌려주는 `claudeAnalyzer`를 만들어 `app.js`의 한 줄만 바꾸면 됩니다. 트래킹·이펙트·추천 UI는 그대로 유지됩니다.

## 이 버전이 주장하지 않는 것

- 얼굴형·무드·스타일 판정이 정확하다 (비율·표정·색 기반 근사치이며 화면에도 "근사치"로 표기)
- 추천이 실제 착용 결과를 보장한다 (공개 자료로 정리한 카탈로그 기준의 규칙 매칭)
- 영상·사진을 수집한다 (모든 계산은 브라우저 안에서만 이루어지고 아무것도 전송·저장하지 않음)
